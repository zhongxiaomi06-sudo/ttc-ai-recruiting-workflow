# TTC 飞书网页助手使用说明

这个助手基于 `BlueSkyXN/feishu-toolkit` 的飞书网页油猴脚本思路做了招聘场景改造，但不是原仓库的完整复制版。

目标链路：

```text
飞书网页版可见群消息/私信
  → 油猴脚本读取 DOM
  → 本地 candidate-collector daemon
  → 简历/候选人解析与打分
  → 生成回复草稿
  → 回填到飞书输入框
```

## 安装

1. 确认本地服务已启动：

   ```bash
   cd /Users/ashley/Downloads/ttc的交易系统/candidate-collector
   ./run.sh
   ```

2. Chrome 安装 Tampermonkey。

3. 打开本地脚本安装地址：

   <http://127.0.0.1:8765/feishu-web-bridge.user.js>

4. Tampermonkey 弹出安装页后点击安装。

5. 打开飞书网页版：

   - <https://www.feishu.cn/messenger/>
   - 或 Lark/飞书其他网页域名。

页面右下角会出现 `TTC 飞书网页助手` 面板。

## 默认行为

- 自动监听当前飞书页面中可见的消息。
- 默认只捕获包含这些招聘关键词的消息：
  - 简历、候选人、面试、招聘、猎头、JD、岗位、BOSS、猎聘、帮我看、评估、推荐、消费、品牌、战略、投后、咨询。
- 识别到候选人资料时，会写入本地候选人库。
- 生成回复后，默认只填入飞书输入框，不自动发送。

## 面板开关

- `启用消息读取`：总开关。
- `自动填入回复草稿`：从本地系统拉取待发送回复，并填入当前飞书聊天输入框。
- `自动发送（谨慎）`：打开后会尝试按 Enter 发送。建议先关闭，稳定后再开。
- `读取所有可见消息`：不按关键词过滤，当前页面可见消息都会发送到本地。慎用。
- `读取选区/当前页`：如果你选中了某段文字，只读取选区；没有选区就读取当前页文本。
- `扫描可见消息`：手动触发扫描。

## 本地接口

- `POST /api/feishu-web/message`：接收飞书网页消息。
- `GET /api/feishu-web/messages`：查看最近读取的消息。
- `GET /api/feishu-web/pending-replies`：油猴脚本轮询待填入草稿。
- `POST /api/feishu-web/reply-ack`：标记草稿已填入/已发送。

## 安全边界

- 只能读取你当前浏览器账号已经有权限看到的内容。
- 不能绕过飞书权限、登录、验证码或管理员限制。
- 默认不会自动发送消息，除非你手动打开 `自动发送`。
- 简历与消息数据只存储在本机 SQLite 数据库。

## 推荐使用方式

先开一个测试群：

1. 打开飞书网页版测试群。
2. 粘贴一段候选人资料。
3. 等右下角面板提示 `已读取消息`。
4. 查看输入框是否出现草稿。
5. 人工确认后再发送。

稳定后再进入正式招聘群使用。
