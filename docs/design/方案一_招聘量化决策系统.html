方案一：招聘量化决策系统

Recruiting Quant OS — 将猎头业务从"人肉经验驱动"升级为"数据+AI驱动的量化投资模型"

量化决策AI Agent 工作流自动化仓位管理 闪电战引擎人机协作

1. 核心类比：主观猎头 → 量化基金

每个顾问一周工作 50 小时 = 50 个单位的"投资预算"。猎头本质上不是"找人"，而是做投资决策——把有限的资金投向哪些公司、哪些职位。

顾问的 5 步投资决策链
  1. 了解公司：新职位出现 → 花 5-10 分钟了解公司背景
  2. 决定是否投入：判断这个客户/职位值不值得花时间
  3. 快速推荐（建仓）：花 1-2 小时搜人，推 3 个人
  4. 等待反馈：推出去后，等待客户反应
  5. 追加投入（加仓）：反馈快、面试率高 → 继续加时间
传统顾问（散户 / 主观投资）量化决策系统（对冲基金 / 量化投资）
顾问自己判断客户好不好系统根据历史数据 + 实时反馈打分
依赖个人经验和直觉规则 + 模型驱动，稳定可复制
难以规模化批量推进标准化职位，地毯式覆盖
一个人盯不了太多职位系统同时监听、评估、执行大量职位
决策靠主观感觉信号回测 + A/B 迭代 + 策略库沉淀

2. 系统定位

核心命题

在有限的人力、时间和数据资源下，如何最大化推荐量 → 面试量 → Offer 量 → 成交量？

这不是一个"AI 搜人工具"，而是一套 招聘量化投资操作系统。核心价值：

  1. 自动发现职位机会
  2. 自动判断职位是否值得投入
  3. 自动搜人 + 生成推荐列表
  4. 自动分配任务给顾问
  5. 根据客户反馈动态加仓 / 减仓 / 止损
  6. 标准化职位批量化、地毯式推进
  7. 人类释放到高价值判断、沟通、谈判

3. 系统适用范围

适合机器处理的岗位
  • 工程师类（后端/前端/测试/数据/算法）
  • 产品经理类
  • AI 产品岗位
  • 年薪 30万-100万 的标准化职位
  • 需求明确、人才池充足的通用职位
  • 客户反馈速度较快的职位
不适合完全交给机器的岗位
  • 高阶管理岗位 / 创始团队核心岗位
  • 极度非标的人才需求
  • 冰山下候选人较多、线上数据不足的职位
  • 需要深度说服、战略沟通、复杂谈判的职位

逻辑：能机器完成的 → 不让人类做；必须人类做的 → 系统派发清晰任务。

5. 分阶段技术路径（MVP → 全量）

Phase 1 · 1-2 个月

职位监听 + 自动识别

让系统"看得见"市场

  • 飞书 Bot 接入群聊，监听消息事件
  • LLM 做 NER + 信息抽取（公司/岗位/薪资/JD）
  • 向量相似度 + 规则做职位去重
  • 新职位 → 自动通知相关顾问
Phase 2 · 1-2 个月

职位评分 + 初始仓位建议

让系统"做判断"

  • 客户评分：历史反馈速度、面试率、Offer率
  • 职位评分：JD完整度、薪资、市场供给、标准化程度
  • 综合分 → 0-5 级仓位建议
  • 判定是否进入闪电战模式
Phase 3 · 2-3 个月

自动 Sourcing + 推荐列表

让系统"动手找人"

  • LLM 解析 JD → 结构化岗位画像
  • 向量检索 + ES 标签过滤 → 搜候选人
  • 多维匹配打分 + LLM 生成推荐理由
  • 任务自动分派给 Sourcing 顾问
Phase 4 · 2-3 个月

反馈追踪 + 动态仓位管理

让系统"会调整"

  • 多渠道信号接入 → 统一反馈事件流
  • LLM 解析非结构化反馈（HR回复/顾问备注）
  • 仓位状态机：自动加仓/减仓/止损
  • 超时未反馈 → 自动提醒顾问
Phase 5 · 持续迭代

全流程自动化 + 学习复盘

系统自主运转，持续进化

  • Temporal/Airflow 编排全流程
  • 定期策略复盘（命中率/策略有效性）
  • 历史成交数据 fine-tune 匹配模型
  • A/B 测试仓位策略，选最优
  • 规则库、标签体系持续更新

7. 职位评分模型

每个职位出现后，不立刻投入大量资源，先做快速评估。类似顾问花 5-10 分钟了解公司。

评分维度

维度权重关键指标
客户质量30%历史反馈速度、面试率、Offer率、合作次数、是否愿意配合
需求清晰度20%JD完整度、薪资竞争力、年限/学历/技能要求是否合理、面试流程是否清楚
人才供给25%内部人才库匹配量、外部平台搜索难度、历史类似职位成功率
成交可能性25%公司吸引力、是否紧急招聘、是否有预算、是否有决策人参与

加仓条件

  • 客户 24h 内反馈 | 候选人进入面试 | 面试反馈正向
  • 客户明确催人 | 需求稳定 | 愿意快速推进
  • 相似候选人命中率高 | 已出现 Offer 机会

减仓条件

  • 客户长期不反馈 | 推荐多人均无面试 | 需求反复修改
  • 候选人普遍不感兴趣 | 薪资竞争力不足 | 面试流程太慢
  • 顾问反馈性价比低

止损条件

  • 推荐超标后完全无反馈 | 客户超过设定期限不回复
  • 职位关闭 | 需求长期模糊 | 投入产出比持续低于阈值

止损 ≠ 永久放弃，而是暂时退出，释放资源给更值得投入的职位。

9. 人机分工模型

角色职责由谁承担
AI 系统监听职位、JD解析、打分、搜人、生成推荐列表、追踪反馈、仓位决策、提醒、数据沉淀、策略复盘机器
Sourcing 顾问补充职位上下文、判断AI结果可用性、打标、反馈原因、优化搜索策略、必要时做触达人类
电话/沟通顾问联系候选人、介绍职位、判断意愿、推进面试、候选人说服、跟进流程、促成 Offer人类
高阶非标顾问高阶岗位 Mapping、创始人/高管沟通、深度说服、Offer 谈判、非标需求拆解人类
策略型顾问分析职位投资价值、设计加仓减仓规则、复盘系统结果、优化 SOP、训练校准 AI人类

早期团队人数不多时，Sourcing 顾问和电话顾问可以是同一个人，但系统设计上必须区分这两个任务类型。

10. 闪电战打法

闪电战核心策略

不是慢慢研究一个职位，而是 快速完成首轮覆盖。目标是让标准化职位在一轮闪电战后就进入"反馈等待"状态。

标准流程：

  1. 系统发现活跃职位
  2. 判断职位是否适合闪电战（标准化程度 + 库存充足）
  3. 自动解析 JD → 搜索人才库 + Boss/脉脉
  4. 按匹配度、可触达性、活跃度排序
  5. 生成第一批推荐列表 → 分发给顾问
  6. 顾问只需快速判断 + 联系
  7. 追踪客户反馈 → 反馈好自动追加 → 反馈差自动减仓

理论上，只要数据源足够，系统可以对大量标准化职位进行地毯式推进，达到一天推荐大量候选人的效果。

11. 数据是系统上限

已有数据资产
  • 客户群里的职位信息
  • CRM / ATS 里的职位和候选人
  • 公司内部人才库
  • 历史推荐、面试、Offer 记录
  • 顾问对候选人的评价
  • 客户反馈速度与合作质量
数据瓶颈
  • 很多职位信息未录入系统
  • 客户真实需求只存在于聊天记录或顾问脑子里
  • 冰山下的高价值候选人不在线上人才库
  • 非标职位难以结构化表达
  • 简历库质量、外部平台数据完整性
  • 历史反馈未标准化沉淀

数据建设优先级： 从 Boss/脉脉获取更多简历 → 清洗完善人才库 → 标准化职位信息 → 标准化候选人标签 → 标准化客户反馈 → 把"口头信息"和"群聊信息"转为系统可用数据。

14. 组织形态变化

关键转变

顾问从"完整流程执行者"分化为系统服务者、校准者、高价值环节执行者。公司能力从"人驱动"变成"系统驱动"。

系统最终价值： 大幅提高标准职位覆盖效率 → 减少低价值重复劳动 → 提高资源配置效率 → 形成规模优势 → 沉淀组织能力（标签、规则、策略、数据、模型、流程）。
