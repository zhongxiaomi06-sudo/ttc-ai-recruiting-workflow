4. 组件说明

4.1 Feishu Toolkit + TTC-Feishu Bridge

保留 Feishu Toolkit 的去水印、解除复制限制、Markdown 复制等能力，并新增 TTC 输出通道：

页面类型提取内容输出
飞书文档 / Wiki标题 + 正文文本（Markdown）JD、公司介绍、人才画像
飞书群聊 / 消息选中或最近消息文本职位线索、客户需求
飞书表格选区文本 / 复制为 Markdown职位列表、候选人名单
飞书妙记 / 会议转写文本（选区）会议纪要、需求要点
部署方式
  1. 安装 Tampermonkey / Violentmonkey
  2. 安装 Feishu Toolkit 正式版（解除复制/去水印）
  3. 安装 ttc-feishu-bridge.user.js，配置本地 Daemon 地址
  4. 在飞书页面点击“发送到 TTC”，或开启自动识别含“JD/招聘/简历”的页面自动推送

4.2 ChatGPT / 链接自动读取器

ChatGPT share link 直接 fetch 会被拒绝（网络层阻塞），因此采用浏览器侧读取：

  • 油猴脚本方式：打开 ChatGPT share 页面后，自动等待内容渲染，提取对话文本，POST 到本地 Daemon
  • Playwright 方式：批量打开 share link，模拟真实浏览器会话，等待 DOM 稳定后提取文本；适用于无用户操作的定时任务
  • 通用 Link Reader：对任意链接，先尝试 Crawl4AI/Firecrawl，失败再调用 Playwright 兜底
边界

只读取用户已拥有访问权限的页面；不绕过登录、验证码、付费墙；ChatGPT share link 读取受限于本地网络可达性。

4.3 本地 TTC Daemon

一个常驻本机的 FastAPI 服务，统一接收、存储、编排所有输入：

端点用途
POST /ingest/feishu接收飞书页面提取的内容
POST /ingest/link接收 ChatGPT/网页链接读取结果
POST /ingest/resume接收 candidate-collector 推送的简历
POST /pipeline/run触发完整工作流：JD → 人才库 → 补全 → 评分 → 输出
GET /api/call-list返回当前待打电话的候选人清单
POST /feedback接收顾问通话/面试反馈，用于学习

4.4 candidate-collector 联动

  • candidate-collector 继续运行在 127.0.0.1:8765，负责 Chrome 扩展读取 BOSS/脉脉/猎聘/Gmail 简历
  • Daemon 定时调用 GET /api/export-jd?min_score=50 拉取已评分候选人
  • candidate-collector 新增 webhook 配置：每次收藏后自动 POST 简历到 Daemon
  • 所有简历 PDF/Word 附件由 MarkItDown 统一清洗后进入理解层

4.5 Playwright 自动点击兜底

当 Crawl4AI/Firecrawl 无法获取页面时，调用 Playwright：

  • 需要登录态的公开页面：使用用户本地浏览器 Cookie/Session
  • 分页列表：自动点击“加载更多/下一页”
  • 动态渲染页面：等待关键 DOM 出现后再提取
  • 可作为定时任务，批量处理链接队列

8. 实施路径

Phase 1 · 1-2 周

本地 Daemon + 飞书桥接

  • 部署 TTC Daemon（FastAPI + SQLite）
  • 安装 Feishu Toolkit + TTC-Feishu Bridge 油猴脚本
  • 实现“飞书页面 → Daemon” 一键推送
  • 实现 ChatGPT share link 的油猴/Playwright 读取
Phase 2 · 1-2 周

人才库 + candidate-collector 联动

  • 对接公司人才库 API（需要接口文档）
  • Daemon 定时拉取 candidate-collector 的 export-jd 接口
  • 建立统一候选人数据模型与去重规则
Phase 3 · 2-3 周

全网补全 + 评分

  • 接入 Crawl4AI + GitHub API + MarkItDown
  • 复用 GoldScoreEngine / TalentMatch 评分能力
  • 输出带证据的排序推荐列表
Phase 4 · 持续迭代

电话清单 + 闭环学习

  • 生成“今日待打电话”清单（含话术、证据、验证问题）
  • 顾问通话后回填反馈，自动校准评分模型
  • 接入 Recruiting Quant OS 仓位管理，形成完整投资闭环
